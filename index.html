SEMANTIC ELEMENTS IN HTML

A semantic element clearly describes its meaning to both the browser and the developer.
In HTML there are some semantic elements that can be used to define different parts of a web page:

THE "BODY" ELEMENT

The "body" tag defines the document's body.
The "body" element contains all the contents of an HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.

THE "MAIN" ELEMENT

The "main" tag specifies the main content of a document.

THE "TITLE" ELEMENT

The "title" element defines the title of the document.
The content of a page title is very important for search engine optimization (SEO)!

THE "HEADER" ELEMENT

The "header" element represents a container for introductory content or a set of navigational links

THE "NAV" ELEMENT

The "nav" tag defines a set of navigation links
Typically the nav element is placed inside the header because the navigation is a part of the header

THE "SECTION" ELEMENT

The "section" element defines a section in a document.
A section is a thematic grouping of content, typically with a heading.

THE "ARTICLE" ELEMENT

The "article" element specifies independent, self-contained content.

THE "FOOTER" ELEMENT

The "footer" element defines a footer for a document or section.

THE "ASIDE" ELEMENT

The "aside" element defines some content aside from the content it is placed in (like a sidebar).

THE "FIGURE" AND "FIGCAPTION" ELEMENTS

The "figure" tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc. The "figcaption" tag defines a caption for a "figure" element. The "figcaption" element can be placed as the first or as the last child of a "figure" element. The "img" element defines the actual image/illustration.
